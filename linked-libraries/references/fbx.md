## Overview

Covers: FBX; Node Wrangler; Basic Usage.

## FBX

FBX (Filmbox) is a common interchange format for 3D — particularly animated characters and rich scenes — and is read by packages such as Autodesk Maya, 3ds Max, and Cinema 4D as well as engines like Unity and Unreal. Its exporter can bake modifiers and animation so what leaves Blender matches what you saw.

Things to watch for. Bone orientation may need fixing: FBX bones read as -X aligned where Blender's are Y aligned, which leaves skinning and animation intact but makes the bones look wrong once imported elsewhere. Animation isn't tied to objects (FBX AnimStacks versus Blender actions), because nothing reliably says which stack is "active" for an object, mesh, or bone — so links are made by hand, and on import an action attaches to the first matching object, bone, or shape key, meaning a batch exported for one object has to be reassigned afterward. Armature instances aren't handled. Importing bone orientation is fiddly and may take some trial and error. Import of animation is bare-bones: curves come in as though baked, i.e. dense keyframes with linear interpolation. If you only want animation, FBX can hold takes by themselves, and you must keep track of which take belongs to which model; the take exported is whatever action is current in the Action editor, so switching off the parts you don't need (and All Actions) trims the file and speeds the round-trip — each action keeps its own name unless you force the lone/current take to "Default Take" (normally left off). Because Blender shades only through nodes whereas FBX assumes a fixed material pipeline, the add-on translates between the two.

It is on by default. The Legacy importer at File ▸ Import ▸ FBX (.fbx) (Legacy) is deprecated — prefer the current FBX Importer. Its Include block offers Import Normals, Import Subdivision Surface (rebuilt as subsurf modifiers), Import User Properties (kept as custom properties), the Enums-as-Strings toggle, and Image Search. Transform offers Scale (against the world origin), Decal Offset, and Manual Orientation with a Forward/Up remap (recall Blender is Y forward, Z up, so Y-up sources need -Z forward, Y up), plus Apply Transform, which folds the space transform into object data to stop stray rotations when target space differs from Blender's — flagged experimental and prone to breaking with armatures or animation. Use Pre/Post Rotation is also present. Under Materials, a name-collision policy chooses Make Unique (each material imported on its own) or Reference Existing (reuse one already named the same). Animation carries an Offset (frames). Armature settings include Ignore Leaf Bones (skip the terminal bone that only marks length), Force Connect Children, Automatic Bone Orientation, and Primary/Secondary Bone Axis.

Exporting (File ▸ Export ▸ FBX (.fbx)), Path Mode governs how references are written, since absolute paths may suit only your machine while relative ones travel better but require keeping files together: Auto (relative within the export folder, absolute outside it), Absolute (full paths), Relative (relative except across Windows drives), Match (mirror whatever Blender uses), Strip Path (filename alone), or Copy (duplicate on export and point at it relatively). Embed Textures is available. Batch Mode emits one file per group or scene; with Group/Scene on, the Current Action option is unavailable (it draws on scene data, and groups aren't bound to scenes) and the group must contain the armature objects for animated actions. Batch Own Directory drops each file in its own folder (handy with Copy Images, so a folder holds a model and its images; this needs a full Python install, without which the toggle is hidden).

FBX export, section by section. Include scopes the export: Selected Objects (only the selection, otherwise the whole scene, except while batch-exporting), Active Collection, Object Types (toggle each type on or off), and Custom Properties. Transform: Scale multiplies the data (10 by default, matching the unit scale most importers expect); Apply Scaling; a Forward/Up remap (Blender being Y forward, Z up); Apply Unit; and Apply Transform, which writes vertices in world space by folding each object's Location, Rotation, and Scale into the mesh first — left off, vertices stay in local object space untouched. Geometry: the Smoothing control writes shading data, with Normals Only (custom split normals, usually the truest when importers accept them), Face (per-face smooth flags), Edge (boundaries inferred from sharp edges), or Smoothing Groups (group membership where same-group faces blend and cross-group faces harden, helpful for tools that expect groups); also here are Export Subdivision Surface, Apply Modifiers (export the post-modifier mesh), Loose Edges, and Tangent Space. Armatures: Primary/Secondary Bone Axis, Armature FBXNode Type, Only Deform Bones, and Add Leaf Bones. Bake Animation: Key All Bones; NLA Strips; All Actions (write every action that fits the chosen armatures, each one's range taken from its own keyframes — off, only the assigned action goes out); plus Force Start/End Keying, Sampling Rate, and Simplify.

Compatibility. The importer is newer and thinner than the exporter: only binary FBX, only version 7.1 and up, and it can't bring in mesh shape keys. Exporting turns NURBS surfaces, Text3D, and metaballs into meshes. Some things simply aren't written, either because FBX lacks them or they're not done yet: instancing (each instanced object is written with its own data rather than shared, and instances appear only in static scenes with animation disabled), certain material textures, vertex shape keys (FBX allows them but this exporter doesn't emit them), animated fluid sims (FBX can't carry them — use the OBJ exporter for a frame sequence instead), and constraints (you get the baked keyframe result, never the constraints themselves).

## Node Wrangler

Node Wrangler bundles a set of tools for working with nodes faster. A lot of them run in every node editor (Compositor, Shader, Geometry Nodes, Texture Nodes), but some are confined to particular editors or behave differently in each; the editor-bound ones carry a label (Compositor, Shader, Geometry Nodes, Texture Nodes), and unlabeled ones apply everywhere. Turn it on under Preferences ▸ Add-ons ("Node Wrangler"). Reach the tools from the node editor's Sidebar panel or with Shift-W for the quick menu; the preferences panel lists the shortcuts, and most tools live in either the sidebar or that menu.

Lazy Connect (Alt-RMB drag, or Shift-Alt-RMB drag): join two nodes without touching their sockets — hold Alt-RMB and drag from one to the other, and it grabs the nodes nearest the drag's ends and links whichever sockets fit best by name, type, and whether they're free. For precision, Shift-Alt-RMB first pops menus of the available inputs and outputs so you choose the exact sockets, which spares you zooming around a big tree. Lazy Mix (Shift-Ctrl-RMB drag): tie two nodes' outputs into a fitting "mix" node — the quick route to selecting nodes and running Merge with Automatic Type Detection.

Merge (Node Wrangler ▸ Merge Selected Nodes): combine the chosen outputs into a "mix"-type node — Mix, Math, Z-Combine, Alpha Over, Mix Shader, Add Shader, or Join Geometry — though Integer, String, and Boolean outputs from Geometry Nodes aren't supported yet. There are automatic routes (the add-on picks the node from the output types) and manual ones (you force a node even when the types and that node don't match). By convention the shortcut's modifier picks the node kind (Ctrl for automatic, Ctrl-Alt for Mix, Shift-Ctrl for Math) and the rest picks the operation (NumpadPlus add, NumpadMinus subtract, NumpadSlash divide, NumpadAsterisk multiply). Automatic detection reads the outputs: a Color output goes to Mix, two Values go to Math, and Add Shader, Mix Shader, or Join Geometry serve their own cases; its modes are Add (Ctrl-=, Ctrl-NumpadPlus — Mix/Math set to Add, or Add Shader for shaders), Multiply (Ctrl-8, Ctrl-NumpadAsterisk), Subtract (Ctrl-Minus, Ctrl-NumpadMinus), Divide (Ctrl-Slash, Ctrl-NumpadSlash), and Mix (Ctrl-0, Ctrl-Numpad0 — Mix node, or Mix Shader for shaders, or Join Geometry for geometry). "Merge Using Mix Node" (Node Wrangler ▸ Merge Selected Nodes ▸ Use Mix Nodes) forces Mix nodes regardless of selection, with shortcuts for Add (Ctrl-Alt-=), Subtract (Ctrl-Alt-Minus, Ctrl-Alt-NumpadMinus), Multiply (Ctrl-Alt-8, Ctrl-Alt-NumpadAsterisk), and Divide (Ctrl-Alt-Slash, Ctrl-Alt-NumpadSlash). "Merge Using Math Node" (▸ Use Math Nodes) forces Math nodes, with Add (Shift-Ctrl-=), Subtract (Shift-Ctrl-Minus, Shift-Ctrl-NumpadMinus), Multiply (Shift-Ctrl-8, Shift-Ctrl-NumpadAsterisk), Divide (Shift-Ctrl-Slash, Shift-Ctrl-NumpadSlash), Greater-than (Ctrl-Comma), and Less-than (Ctrl-Period). "Merge Using Z-Combine Node" (Compositor; Ctrl-NumpadPeriod) links Image and Z-Depth where it can, dropping back to automatic Mix outside the Compositor.

More Node Wrangler tools. Merge Using Alpha Over Node (Compositor; Node Wrangler ▸ Merge Selected Nodes ▸ Use Alpha Over Nodes; Ctrl-Alt-0) merges through Alpha Over nodes, dropping back to the automatic Mix outside the Compositor. Batch Change Blend Mode / Math Operation (Node Wrangler ▸ Batch Change) retunes the blend mode or math op of every selected Mix and Math node together; Alt-Up and Alt-Down step through the previous/next modes, and direct shortcuts exist for Add (Alt-=), Subtract (Alt-Minus, Alt-NumpadMinus), Multiply (Alt-8, Alt-NumpadAsterisk), Divide (Alt-Slash, Alt-NumpadSlash), Greater than (Alt-Comma), and Less than (Alt-Period). Change Mix Factor adjusts the Factor on selected Mix and Mix Shader nodes by shortcut: Alt-Right and Alt-Left nudge it ±0.1, Shift-Alt-Right/Left ±0.01, while Shift-Ctrl-Alt-Left (or Shift-Ctrl-Alt-0) sets it to 0.0 and Shift-Ctrl-Alt-Right (or Shift-Ctrl-Alt-1) to 1.0. Delete Unused Nodes (Node Wrangler ▸ Delete Unused Nodes; Alt-X) prunes the tree of any node that doesn't feed the final result. Swap Links (Node Wrangler ▸ Swap Links; Alt-S) trades the output links of two selected nodes (some connections may be lost if the two have different numbers of linked outputs); with a single node selected it cycles a lone linked input through the available input sockets, swaps the two links when two inputs are linked, or — with more than two — swaps the pair of matching type (such as a Mix node's two Color inputs). Reset Backdrop (Compositor; Node Wrangler ▸ Reset Backdrop; Z) returns the backdrop's position and scale to default. Add Attribute Node (Shader; Header ▸ Add ▸ Input ▸ Attributes) inserts an Attribute node already set to the chosen attribute. Preview Node Output (Shader: Shift-Ctrl-LMB; Geometry Nodes: Shift-Alt-LMB) wires a selected node's output to the tree's final output (the Material Output or World Output in Shader, the final Group Output in Geometry Nodes) so it shows in the viewport, and repeating with the modifier held cycles the available outputs — in Shader any output can go to the final output, but in Geometry Nodes only Geometry outputs can (use a Viewer Node for other types); the related Connect to Output behaves a little differently and works in every node editor. Join Nodes (Node Wrangler ▸ Join Nodes; Shift-P) — see Join in New Frame. Reload Images (Compositor/Shader/Texture Nodes; Node Wrangler ▸ Reload Images; Alt-R) refreshes every image the tree uses without opening the Image Editor. Copy Settings (Node Wrangler ▸ Copy to Selected ▸ Settings from Active; Shift-C) pushes the active node's settings to all selected nodes of the same type. Reset Nodes (Backspace) returns selected nodes' settings to default while keeping their connections. Copy Label (Node Wrangler ▸ Copy to Selected ▸ Copy Label; Shift-V, Shift-C) copies custom labels to the selection — from the active node (Shift-V), from nodes linked to them, or from the names of the sockets they connect to, with Shift-C opening a submenu of all options. Clear Label (Node Wrangler ▸ Clear Label; Alt-L) drops custom labels back to the default node names. Modify Labels (Node Wrangler ▸ Modify Labels; Shift-Alt-L) batch-renames custom labels, adding text front or back and replacing parts. Add Texture Setup (Shader; Node Wrangler ▸ Add Texture Setup; Ctrl-T) attaches a texture node plus Texture Coordinate and Mapping nodes to any shader node — if a texture node is selected it adds only the Coordinate and Mapping nodes, and for a background shader it adds an Environment Texture node.

Node Wrangler, final tools. Add Principled Texture Setup (Shader; Node Wrangler ▸ Add Principled Setup; Shift-Ctrl-T): with a Principled BSDF node selected, choose the command and pick texture files, and it automates adding the Image Texture nodes, loading the images, setting an appropriate Color Space, and wiring outputs into the Principled node. It guesses each texture's role from its file name, and you can edit the matching tags in the add-on preferences; what would otherwise take dozens of clicks collapses to a few. Add Reroutes to Outputs (Node Wrangler ▸ Add Reroutes; Slash) drops a reroute node on every output of the selected nodes. Link Active to Selected (Node Wrangler ▸ Link Active to Selected; Backslash) links the active node to the selection by various rules: "To All Selected" links to every selected node (K, or Shift-K to force-replace existing links); "Use Node Name/Label" links only to selected nodes carrying the same label (apostrophe, Shift-apostrophe to replace); "Use Outputs Names" links only where an output name matches a selected node's name or label (semicolon, Shift-semicolon to replace) — handy for swapping sources at once, e.g. wiring Render Layer outputs to a multi-layer EXR image in the Compositor. Align Nodes (Node Wrangler ▸ Align Nodes; Shift-=) lines the selection up horizontally or vertically at even spacing (similar to scaling on an axis with S X 0 or S Y 0, but evenly distributed). Center Nodes (Sidebar ▸ Node Wrangler ▸ Center Nodes) moves the selection to the editor's center. Select within Frame uses ] to select a frame's direct child nodes and [ to select the selection's direct parent frame. Detach Outputs (Node Wrangler ▸ Detach Outputs; Shift-Alt-D) unlinks a node's outputs while leaving its inputs connected. Add Multiple Images (Compositor/Shader; Add ▸ Input for Compositor, or Add ▸ Texture for Shader) creates one node per selected image — useful for importing render passes or stacking renders. Add Image Sequence (Compositor/Shader; via the same Add menus) builds an Image Sequence node from a single picture in the set, auto-detecting the sequence length and configuring the node; Relative Path makes the file path relative to the current blend-file, and Start Frame is the global first frame of the sequence (assuming the first picture is #1). For reference, the add-on's category is Node, it offers assorted tools to speed up node-based work, it lives at Node editor ▸ Sidebar (or via each tool's shortcut), and it ships GPL-licensed with Blender.

## Basic Usage

Rigify — basic usage. The short path: add a meta-rig from Add ▸ Armature, move its bones to match your character, then click Generate Rig in the armature properties.

Adding a predefined meta-rig (Object Mode; Add ▸ Armature; Shift-A): Rigify packs everything needed to generate complex rig controls and mechanisms into simpler armatures it calls "meta-rigs," and the predefined ones live in the Add menu. The current types are Basic Human (no face or fingers), Basic Quadruped, Human, Cat, Wolf, Horse, and Shark.

Editing bone positions: to fit your character, the meta-rig bones must be repositioned, either in Pose Mode or Edit Mode. Rigify assumes 1 unit equals 1 meter (so a human stands about 2 units tall); if your character is at another scale and you'd rather model than rig, scale it to Rigify's dimensions first — scale up in Object Mode, then bake it with Apply Scale. Human-alignment pointers: keep the legs straight in front view with a slight bend at knee and elbow (Rigify needs to read where they point); keep the spine straight in front view, where the spine's last bone is the head and, top to bottom, the next two count as neck bones (keep those aligned); face bones are fiddly and nearly pointless if you animate the face with shape keys — you can delete them (they sit in the Face bone collection by default; show only that collection, select all, and delete in Edit Mode), and to scale them together, scale the face master bone (placed at the head bone) in Pose Mode.

Pose Mode matching (basic): in the meta-rig's Pose Mode, rotate/scale/translate bones into place, then run Apply ▸ Apply Pose As Rest Pose. Connected bones can't be translated in Pose Mode, so scale parents for general length and refine children. Edit Mode matching (advanced): first set up display — with the meta-rig selected, open Properties ▸ Object, and in the Display panel enable X-ray and set Maximum Draw Type to Wire so bones draw over the geometry; then in Properties ▸ Armature ▸ Display tick Axis to show bone rotation axes while editing.

Generating the rig: with bones placed, return to Object Mode, open the Armature tab, scroll down, and click Generate Rig; it takes a few seconds to a minute depending on complexity and hardware. To tweak, adjust the meta-rig and generate again — Rigify overwrites the existing rig, keeping your modifiers, constraints, and, where it can, previously generated features. If the metarig uses the legacy face rig, an Upgrade Face Rig button appears above Generate Rig to move it to the new modular face system (skinning compatibility is preserved, but existing poses/animations likely won't be, due to subtle control changes). For overwrite to work, keep both the rig and meta-rig visible before regenerating (Rigify unhides them in simple cases but aborts if it can't). As with any Python add-on, Blender's UI freezes until the script finishes, so wait for the rig to appear. Rigify is designed around keeping the meta-rig available for regeneration: deleting it after generating, or heavily editing the generated rig, is discouraged, since it blocks adopting features from later Rigify versions or adapting to Blender's breaking changes (automatic update scripts are provided for meta-rigs when needed, but not for generated rigs).

Binding geometry to a Rigify rig: use whatever skinning tools you prefer, bearing in mind that all deforming bones live in the DEF bone collection; the legacy face's eye and teeth bones don't deform, so bind that geometry with Child Of constraints instead; and armature deform with automatic weights usually does well out of the box given well-placed bones and enough topology. The Blender manual covers bone collections, the Armature modifier, and weight painting in more detail.

Advanced rig generation: the Advanced sub-panel lets you produce several rigs in one scene, update or replace a particular rig, force regeneration of existing widget objects, opt for linked duplicates on the left/right-side widgets, and run a script data-block once generation finishes. The sub-panel is hidden by default (click the Advanced line to open it); Rigify auto-fills some options when they're empty at generation, while others stay fully under your control. Rig Name names a brand-new rig (as opposed to an overwrite); left blank, the new object is named from the metarig — META becomes RIG, metarig becomes rig, and otherwise RIG- is prepended — and when overwriting the Target Rig its name is left alone so you can rename it freely. Target Rig (auto) points at the generated rig to overwrite on the next generation; unset, Rigify makes a new rig object and records it here (and when unset it always creates a new object even if a matching name already exists). Rig UI Script (auto) names the generated script data-block to overwrite, working like Target Rig; that script drives the 3D-Viewport UI for switching bone-collection visibility, changing custom properties, converting IK and FK, and so on. Widgets Collection (auto) names the collection of generated widgets, again like Target Rig. Overwrite Widget Meshes regenerates widgets each time (by default Rigify reuses existing widget objects so you can hand-edit them to fit). Mirror Widgets generates left/right widgets as linked duplicates with negative X scale to flip the right side, enforcing symmetry and cutting the meshes to adjust; when reusing a widget it checks object scale to see whether it was generated mirrored (to avoid re-flipping), so switching an existing character to mirrored widgets means deleting the right-side ones or using Force Widget Update. Run Script executes a Python text data-block after generation, with the generated rig active and selected in Object Mode, to apply custom tweaks — the simplest use being to set generated-constraint properties that have no meta-rig setting (copy the property's full data path via the context menu, paste it into the script, and assign, e.g. `bpy.data.objects["rig"].pose.bones["MCH-spine.003"].constraints[0].influence = 0.6`); doing this by script keeps the change across regenerations. Those familiar with Rigify scripting can import its utility modules and reach the generator via rigify.get_generator(), though since generation has finished, that only reads data produced during it. Library linking: to link a rig into another file, build a collection holding the generated rig and the character mesh (you don't need the meta-rig or the widget collection), link that collection, and run Make Library Override. The rig_ui_template.py text data-block for the rig UI links automatically with the rig, but it won't run until you run it from the Text editor (or save and restart Blender).
