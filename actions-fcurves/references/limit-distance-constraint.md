# Limit Distance Constraint, Limit Location Constraint, Limit Rotation Constraint, Limit Scale Constraint ...

> Blender Actions, F-Curves, and Animation reference. Original prose; identifiers preserved verbatim.

## Overview

Covers: Limit Distance Constraint; Limit Location Constraint; Limit Rotation Constraint; Limit Scale Constraint; Maintain Volume Constraint; Transform Cache Constraint; Transformation Constraint; Troubleshooting; Usage; Editing Keyframes; Keying Sets; Shape Keys Panel; Workflow.

## Limit Distance Constraint

The Limit Distance constraint keeps an object or bone beyond, within, or exactly at a set distance from a target — that is, its location is held outside, inside, or on a sphere centered on the target. Note it has no effect on connected bones, whose position is set by their parent. Options: Target (the object or bone to stay near or away from); Distance (the limit distance, i.e. the sphere's radius, computed automatically when the first target is selected); a Reset Distance button (set it to the current owner-target distance); Clamp Region, how to constrain relative to the sphere — Inside (keep the owner within it), Outside (keep the owner out of it), or On Surface (keep the owner on it); Affect Transform (when the owner is moved by hand, restrict its Location in the Transform panel too, not only its visual position); Target/Owner (the spaces for reading the target position and applying the result); and Influence. Tip: evaluating both owner and target in a Custom Space automatically scales Distance by the space object's/bone's scale.

## Limit Location Constraint

Limit Location keeps an object or bone above, below, or between coordinates along one or more axes. Like other constraints it only considers an object's origin, not its geometry, so a cube whose origin is at its center can still sink halfway into a targeted floor plane. Note it has no effect on connected bones, whose position is set by their parent (see also Floor Constraint). Options: Minimum/Maximum X, Y, Z (number fields set each axis's boundaries and checkboxes set whether each applies — and if a Minimum exceeds its Maximum, the constraint uses the Maximum for both); Affect Transform (when the owner is moved by hand, restrict its Location in the Transform panel too); Owner (the space for determining and limiting the position); and Influence.

## Limit Rotation Constraint

The Limit Rotation constraint limits each axis's Euler rotation angle to a range, clamping out-of-range angles to whichever boundary is nearer on the circle (so with a 0°-90° range and an original 340°, the result is 0°, not 90°). It doesn't work for bones affected by Inverse Kinematics — use the Limit settings in the bone's IK panel instead. Options: Limit X, Y, Z (number fields set each axis's angle range and checkboxes set whether each applies — and even with all limits off, the constraint still removes shear as a side effect); Order (the Euler order used when applying limits, defaulting to the owner's, or XYZ for a non-Euler owner); Affect Transform (when the owner is rotated by hand, restrict its Rotation in the Transform panel too); Legacy Behavior (for backwards compatibility — the semi-broken pre-4.2 behavior that mishandles the looping nature of rotations and causes erratic snapping, occasionally useful with Owner set to Local Space as some old rigs rely on, though drivers on the rotation properties do this better, so new rigs should prefer those); Owner (the space for determining and limiting rotation); and Influence.

## Limit Scale Constraint

Limit Scale caps an object's or bone's scale on each axis with a lower and/or upper bound. Limits of 0 or less won't work correctly and should be avoided. Options: Minimum/Maximum X, Y, Z (number fields set each axis's boundaries and checkboxes set whether each applies — and if a Minimum exceeds its Maximum, the Maximum is used for both); Affect Transform (when the owner is scaled by hand, restrict its Scale in the Transform panel too); Owner (the space for determining and limiting scale); and Influence.

## Maintain Volume Constraint

The Maintain Volume constraint keeps an object or bone at its original volume by automatically scaling down two axes when another is scaled up (and vice versa) — useful for squash and stretch on stylized characters. Options: Mode, how the non-free axes are scaled — Strict (adjust the non-free axes so volume is always exactly kept; scaling the free axis up scales both non-free axes down, and scaling one non-free axis up scales the other down), Uniform (keeps volume correctly only while the object is scaled equally on all three axes; scaling the free axis alone also scales the non-free axes down but in an exaggerated, volume-losing way, so it's not recommended, and scaling a non-free axis has no effect on the other, allowing deviation from the target volume), or Single Axis (like Uniform but correct only while scaling purely along the free axis; since only one axis is adjusted to keep volume rather than three, this can cut the number of animation curves, and changing a non-free axis again has no effect on the other); Free Axis (the axis that keeps its user-set scale, the other two adjusting to maintain volume); Volume (the target volume as a ratio of the original — if all three axes default to scale 2 and that should be kept, set Volume to 2×2×2 = 8); Owner (the space for determining and adjusting scale); and Influence.

## Transform Cache Constraint

The Transform Cache constraint plays back an animation streamed from an Alembic or USD file — for example applying a baked rigid-body simulation to an object. It's added automatically when importing such a file, but applies only to whole-object transforms (location/rotation/scale); for deforming animation the Mesh Sequence Cache Modifier is used instead. Options: Cache File (a data-block menu to pick the Alembic or USD file); File Path (the path to it); Sequence (whether the cache is split into a series of files); Override Frame (use a custom frame to look up cache data instead of the current scene frame, the Frame value being the lookup time or which file in a sequence to use); Frame Offset (subtracted from the current frame for the lookup or file choice); Manual Scale (enlarge or shrink the object relative to the world origin); Velocity Attribute (the Alembic attribute used for motion-blur data, by default .velocities, standard for most Alembic files; Alembic only); Velocity Unit (how velocity vectors relate to time) — Frame (encoded in frames, no FPS scaling needed) or Second (encoded in seconds, scaled by 1/FPS) (Alembic only); Object Path (the path to the Alembic or USD object inside the archive or stage); and Influence.

## Transformation Constraint

The Transformation constraint is the most versatile of the "transform" constraints: it sets an object's or bone's location, rotation, or scale from another's location, rotation, or scale, mixing and matching axes freely — for example driving a gear's X rotation from the Y coordinate of an adjacent rail. It works with per-axis input and output ranges: it first clamps the input to the Map From range, then offsets and scales it into the matching Map To range, so you can map, say, a Y coordinate from (-3m, 3m) to an X rotation from (0, 180°). Options: Target (the object or bone to read the transformation from); Extrapolate (by default the input and output are clamped to the Min/Max, but enabling this lets them exceed those limits — shown by graphs where X is the input with Map From Min 1, Max 4 and Y is the output with Map To Min 1, Max 2); Target/Owner (the spaces for reading from the target and applying to the owner); and Influence. Map From (the transformation read from the Target): Location, Rotation, Scale (the type to read); Mode (for Rotation — the rotation type, including Euler orders, Quaternion, and other Rotation Channel Modes, defaulting to the owner's Euler order, with Quaternion converting channels to weighted angles like the Swing and X/Y/Z Twist modes); and X/Y/Z Min, Max (the input range per axis). Map To (the transformation applied to the owner): Location, Rotation, Scale (the type to apply); Order (for Rotation — which Euler order, defaulting to the owner's); X/Y/Z Source Axis (for each output axis, which input axis it draws from — you can reuse the same input axis); Min, Max (the output range per axis); and Mix, how the result combines with the existing transformation (the choices depending on the transform type) — Replace (the result replaces the existing transformation), Multiply (for Scale — multiply the new values with the existing axis values), Add (for Location/Rotation — add them), Before Original (for Rotation — add the new rotation before the existing one, as if applied to a parent), or After Original (for Rotation — add it after, as if applied to a child). Notes: for historical reasons Mix defaults to Add for location and rotation and Replace for scale; when reading the target's rotation as input, the constraint always wraps the values back into the (-180 to 180) range — e.g. a target X rotation of 420° gives ((420 + 180) mod 360) − 180 = 60 − 180 = −120 — so it's not really suited to transforming an object from a gear's rotation, though rotating a gear from an object's transform works fine; when reading the target's scale, it always uses absolute values (inverting negatives); and when a Min exceeds its Max, both are treated as the Max, so "reversed" mappings aren't possible. Example — a constraint on a gear that sets its X rotation from a rail's Y position: Target = the Rail object; Map From = Location with Y Min -3m, Y Max 3m; Map To = Rotation with X Source Axis Y, X Min 0°, X Max 180°. By default the gear stops rotating once the rail leaves the (-3m, 3m) range, which Extrapolate changes.

## Troubleshooting

Driver troubleshooting. Scripted Expression: Blender restricts Python script execution by default, so a Scripted Expression Driver Type outside the Simple Expressions subset requires opening the file as Trusted Source, or enabling Auto Run Python Scripts in Preferences ‣ Save & Load ‣ Blender Files. Rotational properties are radians: parts of the UI use different units for angles, but in the Graph Editor while working with Drivers all angles are radians.

## Usage

Drivers can be added to properties through their context menu, a shortcut, by copy-paste, or by typing an expression straight into the property's value, and are then usually edited in the Drivers Editor or via the simplified Edit Driver popover from the property's context menu. Add Driver (Context menu ‣ Add Driver; Ctrl-D): the usual way is to RMB a property and choose Add Driver, or hover it and press Ctrl-D — this adds a driver with one (to-be-filled) variable and shows the Edit Driver popover. Edit Driver (Context menu ‣ Edit Driver): a popover to edit the custom expression and input variables without opening the full editor, sufficient since many drivers don't use their F-Curve. Open Drivers Editor (Context menu ‣ Open Drivers Editor): opens a new window with the editor and selects the property's driver. Copy & Paste (Context menu ‣ Copy Driver / Paste Driver): copy and paste drivers via the context menu, saving time when several share settings. Copy Driver to Selected (Context menu ‣ Copy Drivers to Selected / Copy Driver to Selected / Copy All Drivers to Selected): copy the active item's driver for a property to the same property on all selected items where it exists. Copy As New Driver (Context menu ‣ Copy As New Driver): quickly make a driver that sets a property to another's value by using the input property's Copy As New Driver option and pasting onto the output property with Paste Driver, or add the new variable to an existing driver with the editor's Paste Driver Variables button. Expression: a quick way to add a scripted-expression driver — click the property, then type a hash # followed by the expression, e.g. #frame, #frame / 20.0, #sin(frame), or #cos(frame). Removing drivers (Context menu ‣ Delete Driver(s) / Delete Single Driver; Ctrl-Alt-D): remove the driver(s) for a property, either the single selected property/sub-channel or all components of a vector.

## Editing Keyframes

Editing keyframes. Insert Keyframe (Object Mode, Pose Mode, Video Sequencer Preview; Object/Pose/Strip ‣ Animation ‣ Insert Keyframe; I): several ways to add keys — in the 3D Viewport or Video Sequencer Preview, I keys properties per the Default Key Channels preference (or the active Keying Set instead, when one is active); hovering a property and pressing I, or RMB ‣ Insert Keyframe; and with the "Pie Menu on Drag" preference on, holding I and moving the cursor opens a pie menu to insert Location, Rotation, Scale, or Available. Auto Keyframe is the Timeline header's record button, which keys the set frame automatically when a transform-type property changes (see Timeline Keyframe Control). Insert Keyframe with Keying Set (same modes; ‣ Insert Keyframe with Keying Set; K) keys a chosen Keying Set, with a menu of available sets. Delete Keyframes (same modes; ‣ Delete Keyframes…; Alt-I): in the 3D Viewport or Video Sequencer Preview, Alt-I removes current-frame keys from selected objects, bones, or strips; or hover a value and press Alt-I, or RMB ‣ Delete Keyframe. Clear Keyframes (same modes; ‣ Clear Keyframes…; Shift-Alt-I) strips every keyframe from the selected object, bone, or strip. Keyframes are edited in the Graph Editor or the Dope Sheet. Example — keyframe a cube's location, rotation, and scale: set the frame to 1, select the cube in Object Mode and press I in the 3D Viewport (recording location, rotation, scale on frame 1), set the frame to 100, transform with Move (G)/Rotate (R)/Scale (S), and press I again; press Spacebar to play.

## Keying Sets

Keying Sets are a collection of animated properties used to keyframe several properties at once — for example, pressing K in the 3D Viewport lists the available sets, and Blender keys whichever you choose; there are built-in sets and custom ones called "Absolute Keying Sets". Keying Set panel (Properties ‣ Scene ‣ Keying Set): add, select, and manage Absolute Keying Sets — Active Keying Set (a list of the scene's sets, the selected one becoming active), Add Empty Keying Set, Remove Active Keying Set, Description (a short description), and Export to File (export the set to a Python File.py, re-added by running File.py from the Text Editor). Active Keying Set panel (Properties ‣ Scene ‣ Active Keying Set): add properties to the active set — Paths (a list, each with a Data Path to a property to add), Add Empty Keying Set Path, Remove Active Keying Set Path, Target ID-Block (the ID Type and the object IDs data path), Data Path (the rest of the property's path), Array All Items (use all items or pick an array index), and F-Curve Grouping (which group to add channels to — Keying Set Name, None, or Named Group). Keyframing settings: General Override controls all properties in the set, Active Set Override controls individual ones (and the same Preferences settings override these when enabled); common settings are Needed (only insert keyframes where the F-Curves need them) and Visual (insert from the visual transformation). Adding properties to a Keying Set (Context menu ‣ Add All/Single to Keying Set; K): RMB a UI property and pick Add Single to Keying Set or Add All to Keying Set (added to the active set, or a new one if none exist), or hover the properties and press K for Add All. Set Active Keying Set (Shift-K): press Shift-K in the 3D Viewport, select a set in the Keying Set panel, or select one in the Timeline header's Keying popover. Whole Character Keying Set: the built-in Whole Character set keyframes all properties likely animated in a character rig, ignoring bones whose name starts with COR (Corrective), DEF (Deformation), GEO (Geometry), MCH (Mechanism), ORG (Original from meta rig), or VIS (Visualization), assuming these are technical bones not animated directly (the Rigify add-on generates such bones).

## Shape Keys Panel

Shape Keys panel (Properties; all modes; Object Data ‣ Shape Keys) creates, organizes, and manages shape keys in a Tree View where keys can be reordered by click-dragging. Per shape-key element: Value/Frame (in Relative mode, Value is the key's current influence blending between the shape at 1.0 and its reference key — usually the Basis — at 0.0, extrapolatable above 1.0 and below 0.0; in Absolute mode, Value gives the Evaluation Time at which the shape reaches maximum influence); Mute (unchecked, the key is excluded when mixing the stack into the viewport result); and Lock Shape (lock a key against accidental edits — most sculpt and edit-mode tools that move vertices abort with an error if the active key is locked, though operators that change all keys identically, like Apply Object Transforms, and most topology-changing edit-mode operators don't check the lock, since topology is usually finalized before keys exist). Shape Key Specials: New Combined (add a key with the object's current deformed shape, unlike the list's button which always copies the Basis regardless of the mix); Duplicate (copy the active key); New from Objects (add selected objects' vertex positions as new keys, or update same-named keys — select the source object(s) to copy from, then the target, and run it); New from Objects Flipped (like New from Objects but mirroring vertex positions across the local X axis, e.g. making a Left Smile from a Right Smile — all meshes' topology must match); Update from Objects (update the active object's keys from selected objects' same-named keys — select the sources, then the target, and run it); Update from Objects Flipped (a mirrored update across local X, for symmetrical keys with mirrored geometry or pose-corrective shapes); Flip (mirror the shape keys on the X axis in Object Mode, requiring perfectly symmetrical vertices — use Mesh ‣ Symmetrize in Edit Mode); Flip (Topology) (like Flip but detecting mirrored vertices by topology, so the mesh needn't be perfectly symmetrical); Make Basis (make the selected key the new Basis — its vertex positions are applied to the base mesh and all other keys readjusted, useful when a corrective or edited key should become the default shape); Apply to Basis (apply the selected key's deformation to the Basis and remove the selected key, so the result becomes part of the base mesh and remaining keys recalculate relative to it); Apply All (apply the currently evaluated visible shape to the mesh data and remove all keys); Delete All (remove all keys and their effects); Relative (set the keys to Relative or Absolute); Shape Key Lock (pin icon — show the active shape unblended in the viewport, auto-enabled in Edit Mode); Shape Key Edit Mode (edit-mode icon — when on, entering Edit Mode shows and edits the current blend rather than the active key at maximum influence); and Add Rest Position (create a vertex-domain Attribute named rest_position copying the position attribute before shape keys and modifiers are evaluated, mesh objects only). Relative Shape Keys options: with relative keys, the value shown per shape is its current weight in the Mix — Clear Shape Keys (set all weights to zero, to guarantee the viewport result is unaffected by shapes); Value (the blend weight between the key and its reference, 0.0 being 100% reference and 1.0 the key); Range (the min and max for the active key's value, extrapolatable below 0.0 or above 1.0); Vertex Group (limit the active key's deformation to a vertex group, useful for breaking a complex shape into components); and Relative To (the key to deform from — its Reference Key). Note that internally relative keys are stored as mesh-shape snapshots rather than direct offsets, with the relative offsets computed by subtracting the Reference Key from the snapshot.

So replacing the Reference Key subtracts the difference between the new and old reference from the current key's relative deformation. Absolute Shape Keys options (see Relative or Absolute Shape Keys): with absolute keys, each shape's listed value is the Evaluation Time at which it becomes active. Re-Time Shape Keys (clock icon): absolute keys are timed by list order at a constant interval, and this button resets that timing — handy after keys are removed or reordered. Interpolation controls how the shapes interpolate between keys (Linear, Cardinal, Catmull-Rom, B-Spline), where the red line in the figure shows the interpolated values between the black-dot keys. Evaluation Time controls the shape-key influence; scrub to see the current configuration's effect, and this property is typically keyed for animation or driven.

## Workflow

Shape keys are created and managed in the Shape Keys panel of the Properties editor's Object Data tab (the Mesh tab for mesh objects). Edit a key by selecting it in the panel, then dragging the object's vertices to new positions in the 3D Viewport; the panel also has controls to set each key's Value (its influence or weight), letting you view a shape alone or see it combined with other active keys. Adding and removing vertices: you can't add or remove vertices within a shape key — vertex count and connectivity come from the object's topology (mesh, curve, surface, or lattice), and a key only stores existing vertices' positions, so every key always includes every vertex; new vertices are recorded by all existing keys at their creation position, and although you technically can add or delete vertices after creating keys, it's best to finalize the topology first. Adding shape keys: the button next to the list adds a key that's a copy of the Basis, regardless of what's currently visible in the viewport, whereas Specials ‣ New Shape from Mix makes a key from the currently visible vertex configuration (including other keys' influence). For relative-key facial animation, a common technique is to first make a complex extreme pose (such as anger or surprise), then break it into smaller components by applying a temporary vertex group to it and copying with New Shape from Mix, which reduces conflicts between keys that would otherwise double up. Relative shape keys: in Object Mode, add a new key in the panel; Basis is the rest shape and "Key 1", "Key 2", etc. are the new ones; switch to Edit Mode, select "Key 1", and modify the mesh (without adding or removing vertices); select "Key 2" (the mesh resets to Basis), modify it, and repeat for more keys; switch back to Object Mode and adjust the Value sliders to blend — the example shows, left to right, Basis, "Key 1", "Key 2", and a mix of "Key 1" (1.0) and "Key 2" (0.8) (for practical examples, see Combining Shape Keys and Drivers). Absolute shape keys: add a sequence of keys as above, disable the Relative checkbox, click Reset Timing, switch to Object Mode, and drag Evaluation Time to scrub through the sequence — and animating Evaluation Time with a driver or keyframes creates shape-key animations. Edit Mode also offers shape-key operators with extra modeling tools: Blend from Shape and Propagate to Shapes.
