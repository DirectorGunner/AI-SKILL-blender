# Relations, Selecting Bones, Bone Structure, Toolbar ...

> Blender Actions, F-Curves, and Animation reference. Original prose; identifiers preserved verbatim.

## Overview

Covers: Relations; Selecting Bones; Bone Structure; Toolbar; Spline IK; Apply; Clear Transform; Copy/Paste Pose; Flip Quats; In-Betweens; Add IK to Bone; Propagate; Show/Hide; Tool Settings; Selection Sets; Armature Deform Parent.

## Relations

Relations (All Modes; Bone ‣ Relations): this panel manages the bone's relationship with its parent and shows which bone collections it belongs to. Parenting: Parent is a Data ID for choosing the parent bone; Relative Parenting (Pose Mode only) changes how the bone's transform applies to its child objects; and Connected attaches the bone's head to its parent's tail. Transformations: by default children inherit their parent's position (with their own offset), rotation (keeping a constant relative rotation), and scale (again with an offset), just like ordinary child objects, and you can change this per bone in the Relations panel. Local Location, when off, evaluates the location transform in the parent bone's local space rather than the bone's own rest-pose local-space orientation. Inherit Rotation, when off, breaks the rotation link to the parent, so the child keeps its rotation in armature-object space when the parent rotates. Inherit Scale picks which parent-scaling effects the bone inherits, and these inheriting behaviors propagate down the hierarchy (scaling a bone scales its descendants by default), so disabling a bone's Inherit Scale or Inherit Rotation breaks propagation for it and its descendants. Its options: Full (inherit all parent scaling and shear); Fix Shear (apply full parent effects to the child's rest state, then remove shear in a way that preserves the bone's direction, length, and volume and minimally affects roll, before combining with the child's local transform — though non-uniform inherited scale can still reintroduce shear from the child's or its children's local rotation); Aligned (inherit parent scaling as if the child shared the parent's orientation, so parent X scale always maps onto child X scale, and likewise for the other axes); Average (inherit a uniform factor equal to the parent's total volume change); None (ignore all parent scaling and shear); and None (Legacy) (ignore all scaling provided the parent isn't sheared, with no guarantees if it is — replicating the old Inherit Scale checkbox and possibly to be removed). The Inherit Scale options are tools for avoiding shear caused by non-uniform scaling combined with parenting and rotation, with no single best choice: None gives full control of the child's scaling (e.g. to drive it with constraints); Average blocks squash-and-stretch propagation between sub-rigs while letting uniform size changes through; Aligned suits bone chains like tentacles, propagating lengthwise scaling as lengthwise and sideways as sideways however the chain bends (similar to None plus Copy Scale from the parent); and Fix Shear can help at an appendage's base, reallocating squash and stretch between axes by the parent/child rest-orientation difference, behaving closest to Full while suppressing shear. Connected bones can't be moved: since their root must sit at the parent's tip, you can only move the tip if the parent stays put, which rotates the child — so pressing G with a connected bone selected makes Blender switch to rotation. Relationships also shape how multi-bone selections transform: unrelated selected bones transform independently; within one family only the "most parent" selected bones really transform while descendants follow through parenting as if unselected (so a selected, active third tip bone may only shrink through its parent, like the unselected ones); selecting connected and unconnected bones and starting a move affects only the unconnected ones; and if a child connected hinge bone is selected with the most-parent selected bone connected, pressing G does nothing because Blender stays in move mode, which can't move a connected bone. So when posing a chain, work from root to tip — Forward Kinematics (FK) — while Blender's other method, Inverse Kinematics (IK), poses a whole chain by moving its tip.

This relations behavior is somewhat extended by the pose library. Bone Collections: this list shows the collections the bone belongs to — press the eye icon to show or hide a whole collection, the star icon to show only it (and others marked "solo"), and the X icon to remove the bone from that collection; to assign the bone to other collections, use the M or Shift-M shortcuts (see Moving Bones Between Collections) or the Armature properties panel.

## Selecting Bones

You can select and edit armature bones in both Edit Mode and Pose Mode; this covers Edit Mode (Pose-Mode selection is similar, with differences noted in the posing section). As with mesh vertex/edge selection, there are two ways to select a whole bone in Edit Mode: directly by its body, or by selecting both its joints (root and tip) — an important distinction, since selecting joints alone can give non-obvious results about which bone is actually selected. Unlike the mesh display type, the armature display type doesn't affect selection, so you pick a joint or body the same way whatever the visualization. Selecting bone joints: the standard selection methods apply. Inverse Selection — these tools act on joints, not bodies, so Inverse (Ctrl-I) flips the joint selection, not the bone selection; since a bone counts as selected only when both joints are, inverting the joints yields a different set of selected bones. Selecting connected joints: selecting the root of a bone connected to its parent implicitly selects the parent's tip too (and vice versa) — recall that, for joints, a parent's tip is the same point as its children's roots. Selecting bones: click a bone's body to select it (implicitly its root and tip), Shift-click to add or remove, and use relation-based options too — Pick Shortest Path (Ctrl-click) selects the path from the active bone to the one under the cursor. Deselecting connected bones: with several connected bones selected, deselecting one drops its tip but keeps its root if that root is also the tip of another selected bone — for example, Shift-clicking "Bone.003" in a selected chain deselects its tip (shared with Bone.004's root), but since its parent "Bone" stays selected, the shared point (Bone's tip = Bone.003's root) stays selected. Select Mirror (Edit Mode; Select ‣ Select Mirror; Shift-Ctrl-M) flips the selection to the other side. More/Less (Edit Mode; Select ‣ More/Less): More (Ctrl-NumpadPlus) grows the selection to connected bones, Less (Ctrl-NumpadMinus) shrinks it, deselecting bones at each region's boundary. Select Linked (Edit Mode; Select ‣ Select Linked; Ctrl-L) selects every bone in the chain the active bone belongs to, with All Forks also catching bones that branch off. Select Similar (Edit Mode; Select ‣ Select Similar; Shift-G) extends by Children (all descendants), Immediate Children (direct children), Siblings (same parent as active), Length (similar length under a Threshold), Direction (Y axis — bones aligned along their length), Prefix or Suffix (matching name part split by "."), Bone Collection (sharing a collection with the active bone), Color, or Shape (same shape object, in Pose Mode). Select Pattern (Pose & Armature Edit Modes; Select ‣ Select Pattern…) selects bones by name (see Object Select Pattern). Parent/Child ("[", "]") deselects the active bone and selects its immediate parent or a child, while Extend Parent/Child (Shift-[, Shift-]) does the same but keeps the active bone selected.

Pose-Mode selection closely mirrors Edit Mode, with a few differences — chiefly that you select only whole bones, not roots or tips. All (Pose Mode; Select ‣ All; A) selects every selectable bone. None (Select ‣ None; Alt-A) deselects all bones while keeping the active one. Invert (Select ‣ Invert; Ctrl-I) toggles every visible bone's selection. Box Select (Select ‣ Box Select; B), Circle Select (Select ‣ Circle Select; C), and Lasso Select (Select ‣ Lasso Select; Ctrl-Alt-LMB) are the interactive box, circle, and lasso tools. Select Mirror (Select ‣ Select Mirror; Shift-Ctrl-M) flips the selection to the other side. Select More/Less (Select ‣ More/Less): Parent/Child ("[", "]") deselects the active bone for its parent or a child, and Extend Parent/Child (Shift-[, Shift-]) keeps the active bone selected. Select Grouped (Select ‣ Select Grouped; Shift-G) selects by Collection (sharing a collection with the active bone), Color (same as the active bone), Keying Set (bones in the active keying set), Children, Immediate Children, Parents, or Siblings (same parent as the selection). Select Linked (Select ‣ Select Linked; Ctrl-L) selects the active bone's whole chain, with All Forks also catching branches. Select Pattern (Select ‣ Select Pattern…) matches bone names with wild-cards — "*" any string, "?" any single character, "[abc]" any of those characters, "[!abc]" any character not among them (so "*house*" matches anything containing "house" and "floor*" anything starting with "floor") — with a Case Sensitive option and an Extend checkbox to add to the selection rather than replace it. Constraint Target (Select ‣ Constraint Target) selects the bones used as targets by the current selection.

## Bone Structure

A bone has three elements: the start joint (root or head), the body, and the end joint (tip or tail). With that default armature in Edit Mode, the root and tip can be selected and moved just like mesh vertices; together these joints define the bone by their positions, and they each carry a radius property used only by the envelope deformation method. Roll: enabling the Axes checkbox shows each bone's local axes at its tip, where the Y axis always runs along the bone from root to tip — the bone's roll axis. Bone influence: a bone controls geometry when vertices "follow" it, much as a finger's muscles and skin follow the finger bone, which requires defining how strongly a bone influences a given vertex. The simplest approach has each bone affect the geometry within a set range of it — the envelope technique, since each bone controls only the geometry "enveloped" by its influence area. When a bone is shown as Envelope, in Edit and Pose Mode you can see that influence area, which depends on the Distance property and the root and tip radii; these parameters are detailed further in the skinning pages.

## Toolbar

The armature Edit Mode toolbar offers: Select (select or move); Select Box (box drag); Select Circle (circle drag); Select Lasso (lasso draw); Cursor (relocate the 3D Cursor); Move, Rotate, and Scale (the transform tools); Scale Cage (scale via a control cage); Transform (adjust translation, rotation, and scale together); Annotate, Annotate Line, Annotate Polygon, and Annotate Eraser (free-hand, straight-line, and polygon annotation, plus erasing); Measure (measure scene distances); Roll (rotate a bone around its local Y axis); Bone Size; Extrude (make a new bone connected to the last selected joint); Extrude to Cursor (make a new bone between the last selected joint and the cursor); and Shear.

## Spline IK

Spline IK is a constraint that aligns a bone chain to a curve, combining the easy, flexible, attractive shapes of curves with the predictable, well-integrated control of bones — invaluable for flexible parts like tails, tentacles, and spines, and inorganic items like ropes (full settings are on the Spline IK page). Basic setup: despite the name, the Spline IK Constraint is a Forward Kinematics method (normal bone posing) rather than true IK, though it shares traits with the IK Constraint — operating on multiple bones, not usable on objects, and evaluated after all other constraints; should both a Standard IK chain and a Spline IK chain act on one bone, the Standard one wins, but such setups are best avoided as the results are hard to control. To set it up you need a chain of connected bones and a curve: with the chain's last bone selected, add a Spline IK constraint from the Bone Constraints tab; set Chain Length to how many bones (counting from and including the selected one) the curve should influence; and set Target to the controlling curve — the chain is now driven by the curve. Settings and controls (full list on the Spline IK constraint page; this is just the workflow): Roll control — the usual methods of rotating chain bones around their local Y axes still apply (start at the farthest bone and rotate around local Y to set roll from there on), and Copy Rotation constraints also work, with two limits: bones don't inherit a curve's tilt for roll, and there's no automatic dampened-twist inheritance up the chain (consider Bendy Bones instead). Offset controls — enable Chain Offset so the whole chain follows the curve's shape while sitting at an arbitrary point in 3D space; off (the default), the bones follow the curve in its untransformed position. Length control — Y Scale Mode chooses how bones scale lengthwise (stretch to fit the curve, use the pre-IK scaling, or neither), and the curve object's scale also affects the result. Thickness controls — XZ Scale Mode sets how each bone scales on X and Z: None (keeps X and Z at 1.0), Volume Preserve (X and Z become the inverse of the Y/length scale, preserving "volume"), or Bone Original (uses the X and Z the bone would have after normal evaluation); plus Use Curve Radius, which derives X and Z factors from the average radius of the curve points at each bone's joints, letting the chosen mode be tweaked for artistic control. Tips for nice setups: for good deformation, keep the bones roughly equal in length and not too long for a better curve fit, and ideally build them following the curve's rest-pose shape to minimize problems at sharp bends (especially noticeable with stretching off); to control the curve, use hooks (particularly Bone Hooks), one per control point, with only a few control points needed (one per 3-5 bones gives decent control); and the curve type matters little provided a usable path can be drawn from it (one the Follow Path Constraint could also use), depending on how much hook control you need.

## Apply

Apply (Pose Mode; Pose ‣ Apply; Ctrl-A): Pose as Rest Pose makes the current pose the new rest pose (applying the current transforms into Edit Mode), which also resets skinned geometry to its undeformed state, so you'll generally need to skin it again; Pose Selected as Rest Pose does the same for selected bones only; Visual Transform to Pose applies a bone's post-constraint position so the constraints can be deleted while the bones stay put; and Assign Custom Property Values as Default makes the current custom-property values their defaults, so they form part of the rest-pose state during NLA track mixing.

## Clear Transform

Clear Transform (Pose Mode; Pose ‣ Clear Transform): after transforming bones, clear their transforms to return them to rest position. All resets the selected bones' location, rotation, and scale to default; Location, Rotation, Scale (Alt-G, Alt-R, Alt-S) clear them individually — note that in Envelope visualization Alt-S doesn't clear scale but instead scales the selected bones' Distance influence area (also via Pose ‣ Scale Envelope Distance, which only works in Envelope visualization though it's always shown). Reset Unkeyed clears transforms back to their keyframe state, and Only Selected toggles between operating on the selection or all bones.

## Copy/Paste Pose

Copy/Paste Pose (Pose Mode; Pose ‣ Copy Pose / Paste Pose / Paste Pose Flipped; Ctrl-C / Ctrl-V / Shift-Ctrl-V): copy and paste a pose via the menu or hotkeys — Copy Pose puts the selected bones' current pose in the pose buffer, Paste Pose applies the buffer to the posed armature, and Paste Pose Flipped applies the X-mirrored buffer. Key points: it works at the Blender-session level, so you can use it across armatures, scenes, and files, but the buffer isn't saved (lost on quitting Blender); there's only one buffer; only selected bones are copied; on pasting, selection doesn't matter and the pose applies per bone name (so a copied forearm pose goes to the destination's forearm bone, and nothing happens if no such name exists); and what's actually copied is each bone's position, rotation, or scale in its own space, so the pasted result can differ markedly from the original depending on the bones' rest positions and their parents' current poses.

## Flip Quats

Flip Quats (Pose Mode; Pose ‣ Flip Quats; Alt-F) flips quaternion values to reach the desired rotations while keeping the same orientations.

## In-Betweens

In-Betweens: Pose Mode has several pose-editing tools, plus armature-specific options already covered in the editing pages (auto bone naming, property switching/enabling/disabling, and so on). Push Pose from Rest Pose (Pose Mode; Pose ‣ In-Betweens ‣ Push Pose from Rest Pose) is like Push Pose from Breakdown but interpolates toward the rest position, needing only one keyframe instead of two. Relax Pose to Rest Pose (Pose ‣ In-Betweens ‣ Relax Pose to Rest Pose) is like Relax Pose to Breakdown but brings the pose back toward rest, again needing one keyframe. Push Pose from Breakdown (Pose Mode; Toolbar ‣ In-Betweens Tools ‣ Push; Pose ‣ In-Betweens ‣ Push Pose from Breakdown; Ctrl-E) interpolates the current pose toward the next keyframed position. Relax Pose to Breakdown (Toolbar ‣ In-Betweens Tools ‣ Relax; Pose ‣ In-Betweens ‣ Relax Pose to Breakdown; Alt-E) is for keyed bones: when you edit one away from its keyed position, this gradually brings it back, with smaller steps as it nears. Pose Breakdowner (Toolbar region ‣ In-Betweens Tools ‣ Breakdowner; Pose ‣ In-Betweens ‣ Pose Breakdowner; LMB-drag) makes a suitable breakdown pose on the current frame, and can be constrained while active by pressing G/R/S (move/rotate/scale), B (bendy bones), C (custom properties), or X/Y/Z (axes). Blend to Neighbor (Pose ‣ In-Betweens ‣ Blend to Neighbor; Shift-Alt-E) transitions the current pose toward the neighboring keyframes, and needs a keyframe both before and after the current frame.

## Add IK to Bone

Add IK to Bone (Pose Mode; Pose ‣ Inverse Kinematics ‣ Add IK to Bone; Shift-I) adds an Inverse Kinematics Constraint to the active bone, showing a menu to choose the target: Without Target (no target, available only when no other bone or object is selected), Target New Empty Object (an Empty made at the bone's tail and used as the target, again only when nothing else is selected), Target Selected Bone (the selected non-active bone), or Target Selected Object (the selected non-active object — select it before the Armature when entering Pose Mode, or pick it from the Outliner while already in Pose Mode). Remove IK (Pose Mode; Pose ‣ Inverse Kinematics ‣ Remove IK; Ctrl-Alt-I) removes the IK Constraint from the selected bones.

## Propagate

Propagate (Pose Mode; Pose ‣ Propagate; Alt-P) copies the selected bones' current-frame pose onto the keyframes bounded by the Termination Mode, automating copy-and-paste. Termination Mode decides when it stops overwriting keyframes: To Next Keyframe (copy to the first keyframe after the current frame, excluding any on it); To Last Keyframe (replace the last keyframe, making the action cyclic); Before Frame (all keyframes between the current frame and the End frame, best for scripts given the difficulty of setting that frame, though it can be set in the Adjust Last Operation panel); Before Last Keyframe (all keyframes from the current frame until none remain); On Selected Keyframes (apply to all selected keyframes); and On Selected Markers (all keyframes on frames with Scene Markers after the current frame). End Frame sets the upper bound of the affected range (the lower bound being the current frame).

## Show/Hide

Show/Hide (All Modes; Properties ‣ Bone ‣ Viewport Display; … ‣ Show/Hide): you needn't use bone collections to show or hide bones — as with objects, vertices, or control points, use H (hide the selection), Shift-H (hide all but the selection), or Alt-H (show all hidden), or the Hide checkbox in the Bone tab's Viewport Display panel. Hidden bones are per-mode, so bones hidden in Edit Mode stay visible in Pose Mode and vice versa (bones hidden in Pose Mode are also hidden in Object Mode), and in Edit Mode the bone must be fully selected, not just a joint, to hide it.

## Tool Settings

Tool Settings — Pose Options. Auto IK (Pose Mode; Sidebar ‣ Tool ‣ Pose Options ‣ Auto IK) is a quick-posing tool: when on, translating a bone triggers inverse kinematics so the parent, its parent, and so on rotate to follow, with the chain extending from child to parent only where they're connected. While moving, change the chain length (how many bones are affected) with PageUp (+1) and PageDown (-1), or WheelUp/WheelDown; the initial length is 0, meaning follow parent connections as far as possible with no limit, so increasing it the first time sets it to 1 (move only the selected bone) and decreasing from there returns it to 0 (unlimited) — hence you must increase more than once from the start to get a finite length above 1. It's more limited than a configurable IK constraint but handy for quick posing. X-Axis Mirror (Edit and Pose Mode; Sidebar ‣ Tool ‣ Options ‣ X-Axis Mirror) auto-mirrors editing along the X axis: enable it in the Tool tab's Options panel with the armature selected in Edit Mode, and when you have name-paired bones differing only by a side suffix (".R"/".L", "_right"/"_left", …), transforming one also transforms its counterpart through symmetry across the armature's local X axis — an easy way to keep symmetric models (animals, humans) symmetric. Relative Mirror (Edit and Pose Mode; Sidebar ‣ Tool ‣ Options ‣ Relative Mirror) accounts for any relative transformations when using X-Axis Mirror (see Naming bones). Known limitation: Relative Mirror isn't supported with Auto IK enabled.

## Selection Sets

Selection Sets (Pose Mode; Armature ‣ Selection Sets) let you define sets of bones for easy selection while animating, and can be created in local and linked armature overrides. The Selection Set list view shows all sets for the armature, renamable by double-clicking, with a checkbox to the right of each name to include that set when copying to the clipboard. Specials: Delete All Sets (remove every set), Remove Selected Bones from All Sets, Copy Selected Set(s) (to Blender's clipboard), and Paste Selected Set(s) (from it). Assign adds the selected bones to the active set, Remove takes them out, Select selects all bones in the active set, and Deselect deselects them.

## Armature Deform Parent

Armature Deform Parent (Object Mode and Pose Mode; Object/Pose ‣ Parent ‣ Armature Deform; Ctrl-P) is a way to create and set up an Armature Modifier: first select all the child objects the armature will influence, then lastly the armature itself, press Ctrl-P, and choose Armature Deform in the "Set Parent To" pop-up. The armature becomes the parent of all the children, and each child gets an Armature Modifier referencing it (the Object field). With Empty Groups: parenting creates empty vertex groups on the children (where they don't already exist), one named after each deforming bone in the armature, leaving them with no weights assigned; groups are made only for bones set up as deforming (Properties ‣ Bone ‣ Deform), and you then manually select vertices and assign them to a chosen group so the matching bone influences them — choose this when you've already created and weighted the mesh's vertex groups. For example, with bones "BoneA", "BoneB", "BoneC" and a cube, parenting the cube gives it three new (empty) vertex groups of those names. With Automatic Weights: like With Empty Groups, but it fills the groups, computing each bone's influence on vertices from their distance to the bone (the "bone heat" algorithm) and assigning those as weights; it's easier to set up but can deform child objects in unwanted ways, since overlaps can occur when deciding which bones should influence which vertices for complex armatures — the symptom being parts that don't deform as expected when posing, which you then fix by manually editing vertex weights. With Envelope Weights: similar to Automatic Weights, but influences are computed from the Bone Envelopes settings, assigning a weight to each vertex inside a bone's influence volume by distance — so newly included/excluded vertices or new envelope settings aren't picked up unless you reapply Armature Deform With Envelope Weights (or, to have envelope settings take effect instantly, bind the Armature Modifier to Bone Envelopes). Note that if you defined vertex groups with the same names as the skinned bones, both Automatic and Envelope Weights fully overwrite their contents, in which case use With Empty Groups instead (see also Vertex Groups for Bones).
