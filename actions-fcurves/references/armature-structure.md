## Armature Structure

Armatures imitate real skeletons, made of bones that are rigid by default — but you have more freedom than with a real skeleton: besides the natural rotation of bones, you can move and even scale them, and bones needn't connect to each other (they can be entirely free). Still, the most natural, useful setups relate some bones to others, forming "chains of bones" that act like limbs (see Chains of Bones). Chains of bones: an armature's bones can be fully independent (changing one doesn't affect the others), but that's rarely useful — to make a leg, every bone after the thigh should move with it in coordination, which is achieved by parenting each bone to the next, creating a chain. Chains can branch (for example, five fingers on one hand bone). Bones are chained by linking a parent's tip to a child's root, and those can be connected (always at exactly the same point) or free (like an ordinary parent-child object relationship). One bone can parent several children, so it can belong to several chains at once. A chain's first bone is its root bone and its last is the tip bone (not to be confused with the similarly named joints). Chains matter a lot in posing (especially forward kinematics versus automatic inverse kinematics); you build and edit them in Edit Mode, but except for connected bones their relationships don't affect transforms there (transforming a parent won't move its children). The easiest way to manage relationships is the Relations panel in the Bone tab.

## Header

Every constraint has a header: Expand/Collapse (show or hide its settings — a collapsed constraint stays active); the type Icon (red when one or more settings aren't correctly filled in, so the constraint has no effect); Name (initially the type, customizable to something specific); Enabled (turn the constraint on or off while keeping its settings — another way to disable it is setting Influence to zero, which, unlike Enabled, can be animated); and an Extras menu with Apply (Ctrl-A — apply the constraint's result to the owner's own transform, then delete it, with a warning that applying a constraint not first in the stack ignores the ones before it and may give wrong results; see Apply Visual Transform to apply all constraints' combined result without deleting them), Duplicate (Shift-D — make a copy just below this one), Copy to Selected (copy from the active object to all selected objects), Move to First/Last (move it to the top or bottom of the stack), and Delete (X — remove it, losing its settings). A Move handle lets you drag the constraint up or down, which can change the outcome since the stack evaluates top to bottom.

## Stack

The list of constraints affecting an object or bone is the Constraint Stack, evaluated top to bottom — reordering it can change the result drastically. Move a constraint by dragging its handle with LMB.

## Action Constraint

The Action constraint reads the location, rotation, and scale at a given frame inside an Action and applies them to an object or bone. The frame can be given directly or derived from another object's or bone's location, rotation, or scale — for example using frame 0 of the Action when an Empty is at X = 0 and frame 100 at X = 10, so placing it at X = 5 interpolates to frame 50. (Copy Transforms may suffice for a very simple Action, and Drivers allow more complex math relationships without an Action; an obscure alternative use is to reference an Action that animates other constraints' properties, which this constraint then applies to the constraints after it where the names match.) Options: Target (the object or bone used to compute the Action frame — not needed if Evaluation Time is used); Evaluation Time (specify the frame by number rather than a target's transform property, where 0 is Frame Start and 1 is Frame End, and which can be fixed, keyframed, or driven, with Drivers also able to use Custom Properties); Mix, combining the evaluated Action transform with the owner's original (from preceding constraints) — Replace (the Action replaces it), Before Original (Full) (applied before the owner's, as if the owner were a child of the Action with no constraint, causing shear with a non-uniformly scaled "parent" and rotated "child," like Inherit Scale Full), Before Original (Aligned) (prevents shear by scaling the "child" along its own axes, like Inherit Scale Aligned), Before Original (Split Channels) (each channel — location, rotation, scale — computed separately, so the child's location is affected only by the parent's location, not its rotation/scale), and After Original (Full/Aligned/Split Channels) (like Before Original but as if the Action were a child of the owner); and Influence — with a warning that, for technical reasons, modes other than After Original (Full) and (Aligned) may misbehave on parentless objects. Target sub-options: Channel (the transform property and axis used to compute the frame); Target (the space to evaluate that Channel in); Range Min/Max (the Channel values matching Frame Start/End, where Range Max may be less than Min) — note target rotations wrap to -180° to 180°, and negative scales are treated as positive. Action sub-options: Action (the Action to use; actions can be stored as Assets — see Pose Library); Slot (the Action slot to use); Object Action (bone constraint only — by default it finds keyframes for a bone of the same name, but this uses the Action's object keyframes instead; applying a bone animation to a constrained object, or all bone animations to all bones, isn't supported); and Frame Start/End (the Action frames matching Range Min/Max, where End may be less than Start to play in reverse).

## Armature Constraint

The Armature constraint transforms an owner object or bone by the weighted pose transforms of one or more target bones — like the Armature Modifier, which does this for every mesh vertex, except the constraint also uses bones whose Deform option is off. (The Child Of Constraint is an alternative when there's a single parent, which can also be an object.) Options: Preserve Volume (stop the owner shrinking when target bones rotate relative to each other); Use Envelopes (use envelopes to weaken farther target bones' influence — for best results set every bone's Weight to 1.0); Use Current Location, bone constraint only (by default it uses the owner bone's rest location to find nearby envelopes and B-Bone segments; this uses the current location instead, including pose and prior constraints — objects have no rest location, so they always use the current one); Add Target Bone (add a Bones-list entry); Normalize Weights (make the Bones-list weights sum to 1.0); and Influence. Note that, unlike the modifier, the constraint doesn't auto-detect nearby bones — each must be added by hand. Bones sub-options: the list of target bones, each with Target (the armature holding the bone — and the constraint, unlike the modifier, can use bones from different armatures), Sub-Target (the bone's name), a remove button, and Weight (which, with the modifier, would come from a vertex group instead).

## Child Of Constraint

The Child Of constraint transforms an object or bone as if it were the child of another object or bone, its main advantage over a normal parent-child relationship being an animatable Influence, so you can switch parents over time. That Influence can also mix several Child Of constraints into a weighted average of multiple parents (though for bone parents the Armature Constraint may be better). Note it can "parent" one bone to another but isn't well suited to building bone chains — notably it can't emulate connected bones (where the parent's tip always meets the child's base). Options: Target (the parent object or bone); Location, Rotation, Scale (which of the child's axes to affect); Set Inverse (click when the child's transform is right with the constraint off but wrong with it on); Clear Inverse (tell the constraint the child's current transform with it disabled is the correct relative transform, applied as an offset to the parent once re-enabled); and Influence.

## Floor Constraint

The Floor constraint sets up a flat, infinite floor (or wall or ceiling) the owner can't pass through. Like other constraints it only considers an object's origin, not its geometry, so a cube whose origin is at its center can still sink halfway into a targeted floor plane — the Offset helps mitigate that (see also Limit Location). Options: Target (the object or bone defining the floor's location and optionally rotation); Offset (move the virtual floor up or down by a distance, e.g. to keep foot bones above the floor geometry); Min/Max (the axis perpendicular to the floor pointing toward its walkable side — Z makes a floor, X or Y a wall, -Z a ceiling, by default along the global axes); Use Rotation (use the Target's local axes for Min/Max instead of global); Target/Owner (the spaces for evaluating and limiting the coordinates); and Influence.

## Pivot Constraint

The Pivot constraint rotates an object or bone about a point other than its origin — a point relative to the same or another object/bone, or fixed in space. Options: Target (the object or bone used as the pivot, which can be left empty, in which case the pivot is relative to the owner itself or fixed in space); Use Relative Offset (when no Target is set — whether the Pivot Point coordinates are relative to the owner or absolute in the world; with a Target they're always relative); Pivot Point X, Y, Z (the pivot coordinates); Rotation Range (the Euler axis and direction for which the constraint is active) — Always (pivot for every owner rotation), -X/-Y/-Z Rotation (only when that rotation is negative or zero), or X/Y/Z Rotation (only when positive or zero); and Influence.

## Shrinkwrap Constraint

The Shrinkwrap constraint snaps an object or bone to a mesh's surface, like the Shrinkwrap Modifier (which snaps every mesh vertex to another's surface). Like other constraints it ignores the owner's geometry and uses only its origin, so an origin at the object's center clips into the target instead of resting on it — mitigated by Distance. Options: Target (the mesh to snap to); Distance (how far to stay from the surface); Mode, how the snap point is found — Nearest Surface Point (the surface point closest to the owner's original location), Project (project the owner's location along an axis and snap to the first surface intersection — see Project Mode Settings), Nearest Vertex (the closest mesh vertex), or Target Normal Project (the nearest surface point that, projected along its normal, intersects the owner's location — better than Nearest Surface Point but slower, and the normal is interpolated from surrounding vertex normals, which may not match a flat-shaded mesh); Snap Mode, how Distance is applied — On Surface (Distance from the surface on the owner's original side, staying outside if it was outside and inside if inside), Outside Surface (Distance away on the outside regardless), Above Surface (like Outside but along the interpolated surface normal, the others applying it along the line between the owner's location and the snap point), Inside ("shrinks" the mesh by Distance and traps the owner inside, doing nothing if it was already inside the shrunk volume), or Outside ("inflates" the mesh by Distance and pushes the owner out, doing nothing if already outside) — Inside and Outside serving as very crude collision detection that's wrong near sharp corners since they only check the closest face; if Mode is Nearest Vertex this setting is unavailable and the owner simply snaps to the point Distance from the nearest vertex regardless of normals; Align To Normal (align the owner's chosen local axis to the target's interpolated surface normal via a swing rotation, unavailable for Nearest Vertex); and Influence. Project Mode Settings: Project Axis (the axis and direction to project along; with no intersection the location is unchanged); Space (the projection axis's coordinate space); Distance (a cutoff after which projection is deemed failed, 0 meaning no limit); Project Opposite (also project the opposite way along the same axis); and Face Cull — Off (snap to the first face regardless of normal), Front (don't snap if the projection hit a face whose normal points toward the owner, so generally it snaps when inside the mesh), Back (don't snap if the normal points away, so generally it snaps when outside), and Invert Cull (with Project Opposite on and an intersection found that way, use the opposite Face Cull option).

## Clamp To Constraint

Clamp To pins an object or bone onto a Curve, working like this: if no explicit Main Axis is chosen it picks one from the Curve's longest bounding-box side; using that axis it compares the owner's original coordinate to the Curve's minimum and maximum coordinates and remaps it to 0-1; and that remapped value is used as a "curve time" to set the position along the Curve (0 the first control point, 1 the last). Unless the Curve is a perfectly straight line, the owner's coordinate along the Main Axis will likely change, and if it moves the wrong way along the Curve, use Switch Direction to flip the control-point order. Importantly, the owner's coordinate is evaluated in world space while the Curve's bounding box is measured in the Curve's own local space — so if the Curve originally ran -5 to 10 on global X but was moved, rotated, and scaled to run 20 to 90 on global Z, the Main Axis is still X and the owner still needs to move from -5 to 10 on global X to traverse it; for the most intuitive results, keep the Curve at default rotation. (Bézier handles and control-point radii count toward the bounding box; the Follow Path Constraint can both position and orient an owner along a Curve's direction.) Options: Target (the Curve to snap to); Main Axis (the axis for the owner's coordinate and the Curve's min/max); Cyclic (off, the owner stops at the Curve's start/end when leaving the bounding-box range; on, it jumps to the opposite side and runs again — mainly for closed, cyclic Curves); and Influence.

## Damped Track Constraint

The Damped Track constraint points an object or bone toward a target — often called "Look At" or "Aim" elsewhere. "Damped" means it uses a pure swing rotation to minimize roll around the tracking axis, unlike the Track To Constraint, which also applies an "Up" axis. Options: Target (the object or bone to point at); Track Axis (the owner's local axis that should point at the target, typically Y for bones, a negative axis pointing away); and Influence.

## Inverse Kinematics Constraint

The Inverse Kinematics constraint rotates a whole chain of bones (not just one) to follow a target — for example, automatically rotating a character's arm bones to reach a hand position (see Inverse Kinematics). Add it quickly with Shift-I (see Add IK to Bone). Note IK constraints are special in modifying multiple bones, so they ignore their stack position and always run last among the affected bones' constraints; to apply a constraint after IK, copy the resulting transform onto a fresh bone chain first (for instance via Copy Transforms). Options: Target (the object or bone the chain should point toward — uniquely, this constraint works without one, in which case the chain-tip bone moves and rotates freely and only its ancestors are constrained); Pole Target (the object or bone setting the chain's roll — e.g. for an arm, the Target sets the hand position while the Pole Target sets the arm's rotation, i.e. the elbow's position); Iterations (the maximum solving iterations); Chain Length (how many bones the constraint affects, from the owner up its ancestors — 1 rotates only the bone, 2 the bone and its parent, and 0 includes all ancestors to the root); Use Tail (use the bone's tail as the chain's end; unchecked uses its head); Stretch (whether bones may scale to reach the Target, only for bones with an IK Stretch above 0); Weight Position (a checkbox for whether the chain end should match the Target's position, plus a slider for how strongly this chain pulls on its bones versus other IK chains sharing them — in such a "tree," a branch with Weight 1 pulls on the trunk twice as hard as one with 0.5); Rotation (a checkbox for whether the chain end matches the Target's rotation, with a similar strength slider); and Influence. iTaSC Solver: when the armature uses the iTaSC solver, the constraint adds IK Type — Copy Pose (as with Standard IK, the chain tip matches the Target's location and/or rotation), with Axis Reference (the meaning of the Lock axes below) being Bone (the tip positioned/rotated so the Target reads 0 on X/Y/Z in the bone's space) or Target (so the tip reads 0 in the Target's space), and Lock Position/Rotation X/Y/Z (whether the tip must match the Target on each axis — and despite the name these don't lock bones to a fixed pose; for that see the bones' IK properties); or Distance (the chain end stays inside, on, or outside a sphere centered on the Target), with Limit Mode — Inside (stay close), Outside (stay away), or On Surface (an exact distance) — and Distance (the distance to keep).

## Spline IK Constraint

The Spline IK constraint fits a chain of bones to a Curve's shape — well suited to flexible parts like tails, tentacles, and spines, and inorganic objects like ropes. Note IK constraints modify multiple bones, so they ignore their stack position and always run last among the affected bones' constraints; to apply a constraint after IK, copy the resulting transform onto a fresh bone chain first (for instance via Copy Transforms). Options: Target (the Curve whose shape to match); Influence (how strongly it affects the chain). Fitting: Chain Length (how many bones, from the owner up its ancestors — 1 fits only the bone, 2 the bone and its parent, and so on); Even Divisions (off, each bone covers a curve distance proportional to its rest length; on, each covers the same distance regardless); Chain Offset (off, the chain moves to match the curve's position; on, it stays at its original start and mimics the curve's shape from there). Chain Scaling: Use Curve Radius (use the curve control-point radii as extra X and Z scale factors); Y Scale Mode, how bones stretch along the curve — None (reset to rest length), Fit Curve (stretch to cover the whole curve), or Bone Original (keep original length including Pose-Mode scale); XZ Scale Mode, how bones scale along the curve normals (their thickness) — None (reset X and Z to 1), Bone Original (keep the Pose-Mode X and Z), Inverse Scale (X and Z set to the inverse of the Y scale), or Volume Preservation (like the Stretch To constraint); and Use Original Scale (apply Inverse Scale or Volume Preservation atop the Pose-Mode bone scales, as in the Stretch To constraint).

## Stretch To Constraint

Stretch To makes an object or bone aim at, and scale toward, a target, but only along the local Y axis, so it's mainly for bones. It can also do "volume preservation," making the owner thinner as the target moves away and thicker as it nears — useful for squash and stretch on stylized characters — and since it just adjusts the X and Z scales by the Y scale, it works for any object type, not only meshes. Options: Target (the object or bone to stretch toward); Original Length (the rest distance between owner and target at which there's no stretching); a Reset Original Length button (set it to the current distance); Volume Variation (an exponent that exaggerates (>1) or reduces (<1) the volume-preservation thickness change, 0 being the same as Maintain Volume None); Volume Min, Max (lower and upper limits for the thickness factor — with Maintain Volume XZ, the X and Z factors are each bounded to the square root of these, so Volume Max 4 lets X and Z grow to at most twice their value; with X or Z, the chosen axis is bound to these limits, growing to at most 4× while the other doesn't change); Smooth (a higher value slows the thickness change near the limits to avoid an abrupt stop); Maintain Volume, whether to thicken/thin as it contracts/stretches — XZ (scale on both X and Z), X (only X), Z (only Z), or None (don't change X and Z) (see also Maintain Volume Constraint); Rotation, how the owner rotates to track the target with its Y axis — XZ (keep the local X axis near its original orientation), ZX (keep the local Z axis near its original orientation), or Swing (a swing rotation like Damped Track); and Influence.

## Track To Constraint

The Track To constraint points an object or bone toward a target, then rolls so a chosen local axis points up — often "Look At" or "Aim" elsewhere. When the owner sits almost directly above the target, roll about the tracking axis turns unstable, in which case use the Damped Track Constraint instead. Options: Target (the object or bone to point at); Track Axis (the owner's local axis that should point at the target, typically Y for bones, a negative axis pointing away); Up (the owner's local axis that should point up in the world) — and Track Axis and Up must differ, or the constraint stops and its icon turns red; Target Z (bring the owner's Up axis as near as it can to the target's Z axis instead of the global one); Target/Owner (the spaces for the target and owner transforms); and Influence.

## Copy Rotation Constraint

The Copy Rotation constraint forces an object or bone to match a target's rotation, removing shear as a side effect. Options: Target (the object or bone whose rotation to copy); Order (the Euler order used during the copy, defaulting to the owner's); Axis (which axes to copy); Invert (which axes to flip the sign of, so 10° becomes -10° and vice versa); Mix, combining the target's current rotation with the owner's original — Replace (the target's angles replace the owner's), Add (the target's angles are added), Before Original (applied before the owner's, as if the owner were a child of the target with no constraint), After Original (applied after, as if the target were a child of the owner), or Offset (Legacy) (replicates the original Offset checkbox, meant to be like Before Original but broken with multiple axes, hence deprecated); Target/Owner (the spaces for reading angles and applying them); and Influence.

## Copy Scale Constraint

The Copy Scale constraint forces an object or bone to match a target's scale. Options: Target (the object or bone whose scale to copy); Axis (which axes to copy); Power (raise the target's scales to this power, before Offset); Make Uniform (instead of per-axis scaling, apply a single uniform scale to all axes that gives the same volume change); Offset (when on, the target's current scale is multiplied with the owner's original scale from preceding constraints rather than copied over); Additive (with Offset on, add the owner's scales minus 1 rather than multiplying — kept for backwards compatibility, generally pointless and best unused); Target/Owner (the spaces for reading scales and applying them); and Influence. Tips: depending on the settings, Power can give a better-looking result than Influence; and to copy one target axis's scale to all the owner's axes, disable the other axes, enable Make Uniform, and set Power to 3.

## Copy Transforms Constraint

The Copy Transforms constraint forces an object or bone to match a target's location, rotation, and scale together. Options: Target (the object or bone whose transforms to copy); Remove Target Shear (strip shear from the target's transform before mixing, leaving pure translation, rotation, and scale — so a rotated child of a non-uniformly scaled parent, which would otherwise shear, and anything copying it, can be fixed); Mix, combining the target's current transform with the owner's original — Replace (the target's transform replaces the owner's), Before Original (Full) (applied before the owner's, as if the owner were a child of the target with no constraint, shearing with a non-uniformly scaled "parent" and rotated "child" like Inherit Scale Full), Before Original (Aligned) (prevents shear by scaling the "child" along its own axes, like Inherit Scale Aligned), Before Original (Split Channels) (each channel computed separately — equivalent to a Copy Location, Copy Rotation, and Copy Scale all using Offset/Before Original, though slightly different with sheared inputs, and differing from Aligned in that the child's location is affected only by the parent's location), and After Original (Full/Aligned/Split Channels) (like Before Original but as if the target were a child of the owner); Target/Owner (the spaces for reading and applying transforms); and Influence.
