## Animation Editors

Blender's animation editors create, edit, and refine animation, letting you work with keyframes, curves, non-linear actions, drivers, video sequencing, and motion tracking. Each has a role: the Dope Sheet organizes and edits keyframes across many objects and data-blocks; the Graph Editor gives fine control over F-Curves for motion and interpolation; the Nonlinear Animation (NLA) Editor arranges and layers action clips for complex sequencing; the Drivers Editor links properties to expressions for procedural animation; the Movie Clip Editor handles motion tracking, mask editing, and stabilization for animation and compositing; and the Video Sequence Editor (VSE) assembles rendered animation, image sequences, audio, and effects into a finished edit. Together they span quick keyframe tweaks through advanced rigging, motion editing, VFX integration, and shot assembly. This page covers features shared across these editors. Playhead: the blue vertical line marking the current frame; move it by clicking or LMB-dragging in the top scrubbing area, or by Shift-RMB click-and-drag anywhere in the timeline. While dragging it can snap to elements visible in that editor — Seconds, Frames, Markers, Strips, or Keys (only ones shown there; with "Strips" on but dragging in the Graph Editor, nothing happens), and snapping toggles mid-scrub by holding Ctrl. Seconds/frames snapping can use a custom increment (e.g. every third frame), always measured from the scene's first frame and ignoring the preview range, and unlike the others it always snaps to the nearest position regardless of snap distance; when options mix, distance-based elements are tried first and seconds/frames only when nothing is close enough. You can also step it one frame at a time with Left/Right or Alt-Wheel, and jump to the start or end frame (or the preview-range ends, if active) with Shift-Left/Shift-Right. Snapping (Header ‣ Snapping) helps place the playhead precisely: Use Snapping toggles it; Snap Distance is the maximum pixel gap before snapping; and Snap Target picks the elements — Frames (frame intervals), Seconds (second intervals), Markers (timeline markers), Keyframes, or Strips (strip start/end, e.g. in the Video Sequencer) — with Frame Step and Second Step setting the interval between snap points for the Frames and Seconds targets. Playback Controls: a footer region in editors like the Timeline, Dope Sheet, Graph Editor, and NLA Editor with options for playback, keying, auto keyframing, and transport, shared across them for a consistent workflow — controlling how animation previews and syncs to audio, inserting and managing keyframes via keying sets and auto keying, navigating with transport controls, and adjusting frame ranges. Playback properties — Sync: when playback can't sustain the desired frame rate, the actual rate (top-left of the 3D Viewport) turns red, and Sync decides how to cope — Play Every Frame (play them all even if it runs slow), Frame Dropping (drop frames when slower than the scene rate), or Sync to Audio (drop frames to stay in sync with audio). Audio – Scrubbing plays bits of sound as you drag the playhead; Audio – Play Audio, unchecked, mutes everything. Playback – Limit to Frame Range stops the mouse from moving the playhead outside the frame range; Playback – Follow Current Frame pans the view to keep up when the playhead leaves the screen. Play In picks which editors update every animation frame (unchecked ones update only when playback stops, with some exceptions), though starting playback in the Graph Editor, Dope Sheet, or NLA Editor plays all editors regardless — an animator-requested convenience. Show – Subframes displays and lets you set fractional frame values; Set Start/End Frame sets the scene's start/end to the current frame (or the Preview Range's, if active). Keying: the Keying popover holds keyframe-insertion options (its name reflects the active keying set). A Keying Set is a named collection of animatable properties: select one and press I (when not hovering an input field) to key those properties.

With no keying set selected, pressing I keys a default property set (e.g. Location/Rotation/Scale for objects); there are predefined keying sets, and you can build your own in the Keying Sets panel. Insert Keyframes (I) keys the current frame, Delete Keyframes (Alt-I) removes them, and New Keyframe Type sets the type for newly made keys. Cycle-Aware Keying applies special handling when keying trivially cyclic curves, to preserve cycle integrity (most useful while tweaking an established cycle): a key inserted outside the cycle's main time range is remapped back inside it, and overwriting one end key updates the other; additionally, when adding a new curve to an action that has a Manual Frame Range with Cyclic Animation on, the curve is made cyclic with a period matching that range, a check also run before the second keyframe is added to such a curve. Auto Keying: with the record button on, Blender keys the current frame each time you move, rotate, or scale an object or bone in the 3D Viewport (or edit a transform property in the Properties Editor) — one neat use is recording a camera path as you fly through a scene (see Fly/Walk Navigation). It only works for transform properties (Location, Rotation, Scale), so changing, say, a material color still needs a manual key. Its Mode is Add & Replace (add or replace keys as needed) or Replace (only replace existing ones); Only Active Keying Set restricts it to that set (otherwise it keys properties outside it too); and Layered Recording adds a new NLA Track per pass for non-destructive tweaking. Transport Controls set the current frame and drive playback: Jump to Start (Shift-Left), Jump to Previous Keyframe (Up), Rewind (Shift-Ctrl-Spacebar, play in reverse), Play (Spacebar), Pause (Spacebar), Jump to Next Keyframe (Down), Jump to End (Shift-Right), and Jump Backward/Forward by Delta (Ctrl-Left / Ctrl-Right) by a user-set amount, with a menu beside the delta buttons to set that delta. Frame Controls: Current Frame (Alt-Wheel) is the frame shown in the 3D Viewport and the playhead's position; Use Preview Range enables an alternative frame range for focusing on part of the animation, letting you replay a short segment without rewinding or changing the whole scene's range (it affects only the 3D-Viewport preview, not rendering), its bounds shown in dark orange, quickly set and enabled by pressing P and dragging a box and disabled with Alt-P; Start/End give the scene's (or active preview range's) start and end frames.

## Deform

Deform (All Modes; Bone ‣ Deform): this panel sets each bone's deformation options. Turning off the panel-header checkbox stops the bone from deforming geometry at all — overriding any weights it was assigned (muting its influence) and excluding it from automatic weight calculation when the mesh is parented with Armature Deform's "With Automatic Weights" option. Envelope is the most general skinning method, working with all skinnable object types (meshes, lattices, curves, surfaces, texts); it's based on proximity between bones and geometry, each bone having two influence areas shown in Envelope visualization — the inside area (the bone's "solid" part, set by the root and tip radius) and the outside area (the lighter region around it, set by Distance). Envelope Distance defines the volume within which the bone influences a deformed object's vertices, with influence falling off by a quadratic decay with distance. Envelope Weight controls the bone's overall influence in envelope skinning, mattering only for geometry shared by more than one bone (typically at joints) — a higher weight outweighs a lower one, and 0.0 acts like disabling Deform. Envelope Multiply controls how the two deform methods combine when both are on: by default, vertices in at least one vertex group are deformed only by the vertex-group method while "orphan" vertices use envelopes, but enabling this multiplies the bone's envelope-based influence on a vertex by that vertex's weight in the matching vertex group, so the vertex-group method is further weighted by envelopes. Radius Head, Tail set the head and tail radii of envelope bones, within whose volume geometry is fully affected.
